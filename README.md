# Ballycrann

A small touch-based audio game set in an Irish village. The player taps **Begin**, learns a handful of gestures from Mary the guide, then wanders Ballycrann meeting villagers, stepping into the bakery for the smell of bread, picking up a heart-shaped stone at the harbour, reaching for a sprig of rosemary at the bakery door.

The whole game is designed for **VoiceOver off** — the page does its own speaking, in Mary's ElevenLabs voice, so the user's screen reader doesn't fight with the narration.

## Gesture vocabulary

| Gesture | What it does |
|---|---|
| Swipe **left** / **right** | Walk east / west along the high street (or move between two interior states inside a place) |
| Swipe **up** / **down** | Look up / look down |
| **Double tap** (one finger) | Greet whoever's there |
| **Two-finger tap twice** | Step inside an open door, or step back out |
| **Two-finger swipe down** | Bend down and pick something up |
| **Two-finger swipe up** | Reach overhead and take something |

Each interior splits into "by the door" and "deeper in" — swipe right inside to walk further in, swipe left to come back. Two-finger tap twice always exits, from either state.

## Tutorial flow

The tutorial is "A Morning Walk." It introduces every gesture before letting the player wander.

1. **scene1** — swipe right
2. **scene3** — swipe left
3. **scene4** — swipe up
4. **scene5** — swipe down
5. **scene6** — double tap (greet)
6. **sceneInside** — two-finger double tap (step in / out)
7. **scenePickup** — two-finger swipe down (pick up)
8. **sceneReach** — two-finger swipe up (reach for)
9. **scene7 → mini-scene** — "find Pat's lost dog" — walks the player down a lane, plays barks, ends when they two-finger-swipe-down to pick up Spot.
10. **chapterComplete → enterWander** — drop into free-roam.

Each scene calls `practiceGesture(target, introId, vignetteIds, onDone)`, which speaks an intro line, waits for three successful gestures, says one vignette per success, then chains to the next scene.

## Wander mode

Eleven positions along the high street:

```
harbour → bakery → bridie → post → school → brian → chapel → catherine → doctor → pub → james
```

Each has a description, look-up, look-down, and sometimes a person to greet (with three rotating greetings), something to pick up, something to reach for. Seven of the eleven have a **door**; two-finger-tap-twice opens them.

Inside each enterable place, the layout is the same — door state and deeper-in state, each with its own description, look-up, look-down, greet, pickup, and reach. Ambient audio loops underneath Mary's narration while the player is inside.

Items are unique — once you've picked up the chapel's holly berry or the doctor's well-bucket stone, attempting again says "you've already got that one."

## Audio generation

All narration is generated by ElevenLabs and committed to the repo. The runtime never calls ElevenLabs directly — it just loads the pre-rendered MP3s.

**To regenerate:**

1. **GitHub → Settings → Secrets and variables → Actions** — ensure `ELEVENLABS_API_KEY` is set.
2. **Actions tab → Generate audio → Run workflow.**
3. Inputs:
   - `force` — tick to re-render lines whose MP3 already exists.
   - `only` — string prefix filter. `t-mini` matches `t-mini-stage1`, `t-mini-stage2`, etc. `w-bridie` matches just that place's lines. Leave blank to consider all.
4. Workflow generates any missing (or matched, if `force`) line and commits the MP3s back to `main`.

Voice ID and model are pinned in `scripts/generate-audio.mjs` at the top — change there if needed.

The page itself plays MP3s lazily: only the current scene + nearby content is preloaded, so initial load is small (~50 KB of audio) and the rest streams in as you play.

## Adding a new line

1. Add the entry to `audio/lines.json` with a stable ID.
2. Reference it in `index.html` via `speakLine('your-id')`.
3. Run the workflow with no inputs — it will pick up the new ID, generate the MP3, and commit.

## Adding a new ambient interior sound

1. Drop the file at `audio/amb-PLACEID.mp3` (e.g. `amb-pub.mp3`).
2. The runtime auto-loops it on entry. Tune the volume + loop behaviour in the `ambientConfig` object near the top of the audio section in `index.html`.

## Deploy

Pushing to `main` automatically deploys via GitHub Pages. The site URL is set in the repo's **Settings → Pages**.

## Built with

- HTML + CSS + vanilla JS, no framework
- Web Audio API for synth tones, ambient loops, and ElevenLabs voice playback
- `speechSynthesis` as a graceful fallback when an ElevenLabs MP3 is missing or hasn't loaded yet
- ElevenLabs for voice (`eleven_multilingual_v2`)
- freesound.org for ambient field recordings
- GitHub Actions for the audio-generation pipeline
- GitHub Pages for hosting
